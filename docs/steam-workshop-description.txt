Play Timberborn with an AI partner. Timberbot API adds external read/write HTTP access to game systems that Timberborn's built-in automation (levers, sensors, adapters) doesn't cover. Colony-wide decisions, building placement, worker management, map analysis, and AI-driven strategy on top of the game's own logic circuits.

[code]
Open the in-game Timberbot widget -> Settings -> choose Claude or Codex -> Start
[/code]

The preferred way to use Timberbot with AI is the in-game Timberbot widget. Configure the agent in the game, launch it with Start, and talk to Claude/Codex in the terminal session Timberbot opens for you. The `/timberbot` skill and prompt/docs workflow still exist for anyone who wants that setup too.

Works with Claude Code, OpenAI Codex, ChatGPT, or any AI agent that can make HTTP calls. Or use it as a scripting API for your own tools, dashboards, and external automation.

[h1]In-game Timberbot UI[/h1]
[b]Corner widget:[/b] a draggable green Timberbot widget appears in the bottom-right corner of the screen. It shows agent status and has:

- [b]Start[/b] launch the agent with your current settings
- [b]Stop[/b] stop the currently running agent session
- [b]Settings[/b] open the full settings modal

[b]Settings modal:[/b] configure the agent directly in-game:

- [b]Binary[/b] choose Claude, Codex, or type a custom binary
- [b]Model[/b] pick from presets or type a custom model
- [b]Effort[/b] pick from presets or type a custom effort value
- [b]Goal[/b] tell the agent what to work toward
- [b]Startup settings[/b] debug endpoint, HTTP port, webhooks, write budget, terminal wrapper, python launcher

Settings persist automatically. The widget position is saved too.

[h1]What the AI can see and do[/h1]
[b]Every beaver and bot:[/b] position on the map, which district they belong to, all 38 wellbeing needs by category, what they're carrying, workplace, contamination status. Bot energy, grease, and deterioration.

[b]Every building:[/b] workers, power state, production progress, inventory, recipes, construction status, effect radius. Place and demolish buildings with full game-native validation. Smart placement finder returns entrance coordinates, path connectivity, power adjacency, flooding status, and water depth for pumps.

[b]The whole map:[/b] terrain height, water depth, badwater contamination, soil moisture, every occupant on every tile (including vertical stacking). ASCII map with height shading.

[b]Colony management:[/b] worker assignment, priorities, crops, tree marking, science unlocks, floodgates, stockpiles, distribution, breeding, clutch control, work schedule, game speed.

[b]A* pathfinding:[/b] place_path builds roads between any two points. Routes around buildings, ruins, water, and terrain obstacles. Auto-places stairs and platforms across elevation changes. Handles diagonal routes, multi-level jumps, and reuses existing paths and stairs. Optional sections param for incremental building across cliffs.

[b]Power networks:[/b] which buildings share a power grid, supply vs demand per network, generator output.

[b]One-call snapshot:[/b] entire colony in a single request. Resources, weather, drought countdown, alerts, and projected days of food, water, logs, planks, and gears.

[b]Brain:[/b] One command gives the AI everything: live colony summary (faction, DC per district, resources, building counts by role, nearby tree and food clusters) plus persistent state (goal, tasks, named maps). Summary is always fresh; goal, tasks, and maps survive between sessions in compact toon format.

[b]Webhooks:[/b] 68 push notification events -- drought start/end, beaver death, construction complete, game over, and more. Your scripts react instantly instead of polling.

[b]Server-side pagination & filtering:[/b] list endpoints default to 100 items per page. Filter by name or proximity without downloading everything. Works with any faction automatically.

[b]Structured error codes:[/b] machine-readable "code: detail" format with context (which building is blocking placement, science costs). Separate crops and trees endpoints for targeted queries.

[b]Output formats:[/b] all endpoints support toon (compact, AI-friendly) and json (full nested objects) via format parameter.

[h1]Live dashboard[/h1]
[code]
timberbot.py top
[/code]
Real-time colony overview in your terminal. Population, resources, weather, drought countdown, wellbeing breakdown, alerts -- all updating live. No browser needed.

[h1]How the built-in agent works[/h1]
Timberbot API runs an HTTP server (port 8085) inside the game. When you press Start in the widget, Timberbot gathers current colony state, uses the shipped Timberbot guide as the instruction file, and launches Claude, Codex, or a custom binary interactively. You can then guide that session in the terminal it opens.

The Python client is included too:
[code]
timberbot.py brain                                # full colony picture: DC, faction, resources, clusters, maps, tasks
timberbot.py top                                  # live colony dashboard
timberbot.py beavers detail:full                  # every need on every beaver
timberbot.py power                                # power networks with supply vs demand
timberbot.py map x1:110 y1:130 x2:130 y2:150 name:base  # ANSI map saved to memory
timberbot.py place_building prefab:FarmHouse.IronTeeth x:120 y:130 z:2 orientation:south
timberbot.py place_path x1:0 y1:0 x2:200 y2:180    # A* path with auto-stairs
timberbot.py launch settlement:Potato save:Tomato       # Windows: auto-launch, macOS: prepare autoload then open Timberborn manually
timberbot.py crops                                  # crops in the ground with growth status
timberbot.py register_webhook url:http://localhost:9000/events events:drought.start,beaver.died
[/code]
Works with any language that speaks HTTP. Python client is optional. Raw curl works too.

[b]OpenAI Codex:[/b]
Point Codex at the mod folder and give it the docs. It can call the HTTP API directly, or you can use the in-game widget and let Timberbot launch it for you.

The mod ships with docs that any AI agent can use. The local docs live in Documents/Timberborn/Mods/Timberbot/docs. The AI reads the single guide first, then uses the API reference only when it needs exact command or error details.

Works with ChatGPT too. Paste docs/timberbot.md as the prompt and keep docs/api-reference.md nearby for exact command details. If someone cannot access the workshop copy, the GitHub repo contains the same docs.

[h1]Getting started[/h1]
1. Subscribe and enable in Mod Manager
2. Start a game. The API starts automatically on port 8085
3. Open the Timberbot widget in the bottom-right corner
4. Click Settings, choose your agent options, then click Start

[h1]Settings[/h1]
The in-game Settings modal is the primary way to configure Timberbot. All settings persist to settings.json automatically.

If you want the manual/advanced path, you can also edit settings.json directly in your mod folder (Documents/Timberborn/Mods/Timberbot/):
[code]
{
  "debugEndpointEnabled": true,
  "webhooksEnabled": true,
  "webhookBatchMs": 200,
  "webhookCircuitBreaker": 30,
  "webhookMaxPendingEvents": 1000,
  "httpPort": 8085,
  "writeBudgetMs": 1.0,
  "terminal": "",
  "pythonCommand": ""
}
[/code]

Some runtime settings may require reloading the save or mod to fully apply.
